# Информационная система на Java Spring Boot

Данный проект представляет собой серверное приложение на стеке Spring Boot 3.x, Hibernate/JPA и PostgreSQL. Приложение предназначено для работы с базой данных в локальном окружении или через контейнеризацию.

## Технические требования

Для успешного запуска проекта необходимо наличие установленного ПО:
*   Java Development Kit (JDK) версии 17 или выше
*   Система сборки Maven или Gradle
*   СУБД PostgreSQL 14+ (для локального запуска без Docker)
*   Docker и Docker Compose (для контейнеризированного запуска)

## Конфигурация окружения

Приложение использует переменные окружения для защиты чувствительных данных. Убедитесь, что следующие параметры определены в вашей системе или в файле `.env`:

*   `POSTGRES_DB` — имя базы данных
*   `POSTGRES_USER` — имя пользователя БД
*   `POSTGRES_PASSWORD` — пароль пользователя
*   `SPRING_DATASOURCE_URL` — URL подключения (например, `jdbc:postgresql://localhost:5432/db_name`)

## Запуск приложения

### Локальный запуск (через Maven)
1.  Убедитесь, что PostgreSQL запущен и создана целевая база данных.
2.  Экспортируйте переменные окружения в терминале:
    ```bash
    export POSTGRES_USER=your_user
    export POSTGRES_PASSWORD=your_password
    ```
3.  Выполните команду сборки и запуска:
    ```bash
    mvn spring-boot:run
    ```

### Запуск через Docker Compose
Для автоматического развертывания базы данных и приложения используйте Docker Compose:
1.  Перейдите в корневую папку проекта.
2.  Запустите контейнеры:
    ```bash
    docker-compose up -d
    ```

## Решение проблем (Troubleshooting)

### Ошибка: "password authentication failed for user "${POSTGRES_USER}""
Если в логах отображается именно такой текст (с символами `${...}`), это означает, что Spring Boot не смог разрешить переменную окружения и воспринял её как обычную строку.

**Решения:**
1.  **В IntelliJ IDEA:** Проверьте `Run Configuration` -> `Environment Variables`. Добавьте туда пары ключ-значение.
2.  **В терминале:** Убедитесь, что вы выполнили `export` переменных в той же сессии терминала, где запускаете проект.
3.  **Значения по умолчанию:** Укажите дефолтные значения в конфиге, чтобы избежать пустых полей: `${POSTGRES_USER:postgres}`.

### Проверка подключения к БД
Чтобы убедиться в работоспособности базы, выполните:
*   Проверку доступности порта: `telnet localhost 5432` (или 5433, если порт переназначен).
*   Прямое подключение через psql: `psql -h localhost -U your_user -d your_db`.
